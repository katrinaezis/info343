Summary

About 3 weeks intothe quarter, I will put you into teams begin working on this challenge. For your final project, each team can decide what they want to build. You can build a new web site for a real client such as a student organization, an artist, a non-profit, or a local business. Or you can build a more sophisticated web application, either for a client or yourselves. It's up to you, but there are a few base-level requirements that I want you to meet (see the full project description

Proposal Requirements

Project Description

This is a written, non-technical description of what you will build. Depending on the specifics of your project, you should answer (most) of the following questions:

  • Who is the client or target audience of the web-page?
  • What is the purpose of your project? What information will it communicate? What will it allow people to do?

Technical Description

This section of your proposal should include a list of the libraries/packages you plan on using to complete your project. This includes, but is not limited to:

  • The JavaScript framework you will use to structure your code (i.e., Angular.js, React.js, etc.)
  • The service you will plan to use for data storage (i.e., Parse, Firebase, etc.)
  • The new technical skills you will need to learn to complete the project

Wireframe

Creating a low-fidelity representation of your webpage is a crucial part of the design process. By laying out the different components of your project and the connections between them, you are able to identify the steps necessary to be successful. These can be handdrawn sketches, powerpoint/keynote presentations, or build using a wireframing tool like balsamiq, Lucid Chart, or another tool of your choice. Regardless of the tool, your wireframe should include the following:

  • A visual representation of each component of your project
  • Descriptions of interactivity (i.e., hover/click/scroll events)
  • Descriptions of interactivity (i.e., hover/click/scroll events)

Note: your wireframe document is not binding, and you'll likely make changes as you actually build the project. However, it should help you elicit important design questions at an early stage in the process
