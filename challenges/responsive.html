One of the most common web development projects is a marketing site for an organization or product. Most every organization and product has one, and it's amazing how many of them are really terrible. Many of them have ugly or distracting visual design, or poor organization of their content. And most look terrible on mobile devices with small screens.

This is a shame because it's just not that hard to build a well-designed, responsive marketing site. In this challenge you will build one of these sites for a fictitious coffee shop named Dawg Coffee. This site will respond to the size of screen it is rendered on, and adjust its layout and styling accordingly.

Objectives

By completing this challenge you will practice and master the following skills

  • Writing semantically-rich HTML pages by choosing the most appropriate elements to wrap around your information
  • Writing well-formatted HTML source that is properly-indented and easy to read
  • Creating hyperlinks that tie pages together
  • Using HTML character entities to represent reserved or difficult to input characters
  • Creating a stylesheet and linking it to your web pages
  • Writing style rules and using styling properties to control the formatting of your pages, including fonts, colors, links, backgrounds, borders, padding, margins, and layouts
  • Using various CSS units, including percentages, pixels, ems, hex colors, and rgba() colors
  • Adding the Bootstrap CSS framework to your web pages to automatically improve the graphic design
  • Using the Bootstrap grid system to create multi-column page layouts
  • Using media queries to adjust page layout and other formatting based on the screen size
  • Implementing a page based on screenshots you would get from a graphic designer

Build and Style the Home Page

In the /dawg-coffee subdirectory, create a new file named index.html. This will contain the contents of the site home page.

Then create a subdirectory named css and inside that directory, create a file named main.css

Basic Page Structure

As you learned in class, open index.html in your editor and build a basic web page. Remember to include the following:

  • The HTML5 doctype declaration
  • An element telling the browser that this page is saved in the UTF-8 text encoding
  • An element telling the browser to set the viewport width to the device width, and the initial scaling to 1
  • An element declaring the page title to be Dawg Coffee
  • An element declaring that the page's icon is the page-icon.png located in the img subdirectory.
  • An element that adds the main.css stylesheet you created in the css directory to the page.

Now that you have the basic skeleton, you can go on to create and style each portion of the page.

Base Page Styles

You will be building this page in a "mobile first" approach. The base style rules will be the ones that should apply to small phone screens, and you will then add media queries to change those rules for larger screens.

In your main.css file, add the following:

  • A directive to import the Bootstrap CSS file from its content delivery network (CDN) location https://maxcdn.bootstrapcdn.com/bootstrap/3.3.5/css/bootstrap.min.css. You only need the CSS file, not the JavaScript file.
  • Directives to import the Google fonts Raleway and Lato. You will need the 300 and 400 weight normal face of both, and the 300-weight italic face of Lato.
  • A directive to import the Font Awesome font from it's CDN location https://maxcdn.bootstrapcdn.com/font-awesome/4.4.0/css/font-awesome.min.css

Then add some style rules that accomplish the following formatting:

  • The default font should be Lato, with Helvetica Neue, Helvetica, and sans-serif as the fall-backs if the browser cannot reach the Google fonts server.
  • The default font size should be 16px and the default weight should be 300.
  • The background color for the page should be #E9EAEE.
  • All headings should use the font Raleway, with Helvetica Neue, Helvetica, and sans-serif as the fall-backs.
  • All second-level headings should be in the color #BC6026.
  • On screens 768 pixels and wider, all second-level headings should be center aligned, and have a bottom margin of equal to the height of the current font.
  • All <svg> elements should be the same height as the current font.

Navigation Bar

Add the HTML elements for the navigation bar. Use the appropriate semantic elements for the entire navigation bar, and the unordered list of links.

For the icon on the left, use the <svg> content that is in the svg/brand.svg file. Copy and paste this into your HTML where it should appear, so that we can apply style rules to it.

The links in the navigation bar should take the user to the following locations:

  • Coffee Cup Icon: the top of this page
  • Cafe: the top of the cafe section on this page
  • Menu: the top of the menu section on this page
  • Story: the top of the story section on this page
  • Order: the order.html page in the same directory as this page (you will create this later)

Now create style rules in your main.css file to accomplish the following

  • The whole navigation bar should be fixed to the top of the viewport so that it doesn't scroll with the page.
  • Background color should be #181C22 by default, but should switch to rgba(24, 28, 34, 0.8) on screens 768 pixels and wider.
  • The text color should be pure white.
  • The font size should be 16 pixels.
  • There should be 15 pixels of space between the edge of the screen and the first link (the icon).
  • The list items should layout in a line as pictured, and not stack on top of each other like list items normally do.
  • There should be 0.5em of padding all around each of the links. The left and right padding should increase to 1em on screens 768 pixels and wider.
  • The links text should be rendered in small capitals regardless of what case was used in the HTML.
  • The links in this navigation bar should not have an underline like they normally do.
  • When the links in this navigation bar get focus, or when you hover over them with the mouse, they should not have an underline, and their color should change to #D87E28. This should also occur for the SVG icon—note the CSS property that controls the fill color for SVG elements is different from the one that controls text color!

Header Area

In your index.html page, add content for the header area. The 'Dawg Coffee' should be a top-level heading. Motto shown on larger screens is just a normal paragraph. Both of these should be inside a Bootstrap container so that they have a bit of padding and a maximum width.

Now add style rules to your main.css file to accomplish the following

  • The header area should have 3em of padding on the top, so that the text is visible below the fixed navigation bar
  • On screens 768 pixels and wider, the header area should use the file background-sm.jpg in the img directory as the background image. The image should cover the entire header area, and not repeat.
  • Also on this size of screen, the text in the header area should be pure white and aligned right.
  • Also on this size of screen, the header area should have a minimum height of 550 pixels, so that more of the image is visible.
  • On screens 992 pixels and wider, the background image for the header should be background-md.jpg. The 'Dawg Pizza' text should also be increased in size to 4em
  • On screens 1200 pixels and wider, the background image for the header should be background-lg.jpg.
  • The motto should not be displayed by default, but should appear on screens 768 pixels and wider.

Headline Section

Below the header, add another section for what we might call the headline (see images above). This should contain the icon and a paragraph with the pictured text. The SVG for the icon is in the file svg/brand.svg. Just as with the header, these elements should be inside a Bootstrap container so that they get some padding and are constrained in width.

Add style rules to accomplish the following:

  • The text in this section should be in italics.
  • The SVG icon should be hidden by default, but should appear on screen sizes of 768 pixels and wider. When it is shown, it's size should be 1.5em and it should have a bottom margin of 1em.
  • Also on screen sizes of 768 pixels and wider, the font size should grow to 1.25em, the text should be center aligned, and the padding on top and bottom should increase to 3em. This will give it a more spacious feeling, drawing the eye towards it.

Cafe Information Section

Next add another section for information about the cafe: its hours, location, and contact information. This section should have a responsive multi-column layout. By default the hours, location, and contact information blocks should stack on top of each other in a single column. But on screens 992 pixels and wider, the layout should switch to three adjacent columns. Use Bootstrap's grid system to achieve this effect. Remember that a Bootstrap grid should always be inside a Bootstrap container.

The SVG icons above each block of information are in the files svg/hours.svg, svg/location.svg, and svg/contact.svg.

In the hours block, the dash between the start and end times should be an 'en-dash', which is slightly longer than a hyphen, and is the appropriate punctuation to use when indicating a range between values.

The map image in the location block is in the file img/map.png. It should also be a hyperlink so that when the user clicks the map image, it opens a new browser window/tab and navigates the browser to the URL https://www.google.com/maps/place/4214+University+Way+NE,+Seattle,+WA+98105.

The contact information block should list the email address, phone number, and Twitter handle for the cafe. The email address should be a hyperlink that when clicked, opens a new email message with that address in the To field. The phone number should be a hyperlink that when clicked on mobile phones, prompts to start a new phone call to that number (on other devices, it might invoke other telephony software if it has been configured). The Twitter handle should be a hyperlink that when clicked, opens a new tab and navigates the browser to the cafe's Twitter page at https://twitter.com/dawgcoffee.

The icons next to the email address, phone number, and Twitter handle should come from the Font Awesome font that you imported into your stylesheet.

After you've added the necessary HTML elements, add style rules to accomplish the following:

  • All text in this section should be center aligned (note that Bootstrap also defines a style class that center-aligns text, so you may just add that style class to your section instead).
  • The background color of this section should be #dbdce3.
  • The map image should have rounded corners with a radius of 5 pixels.

Menu Section

Next, add a section for the menu. Like the section above, this should have a responsive multi-column layout. By default the Drinks and Treats blocks should stack on top of each other, but on screens 768 pixels and wider, they should appear side-by-side.

The menu items are pairs of names and descriptions, so use the most semantically-appropriate elements for these. That is, use elements that represent a list of name and description pairs, and not just a simple unordered list.

Note that the ampersand in the first item under Treats is a reserved character in HTML, so to embed this in the page, you must use its corresponding HTML character entity.

Now add style rules to accomplish the following:

  • The descriptions of each menu item should be in italics, and should be shifted right by 1em.

Our Story Section

Next, add a section for the organization's story. The content in this section should be wrapped in a Bootstrap container so that it has the same padding as the other sections, and has a constrained width.

The image in this section is in the file img/story.jpg. The rest of the text is runs of Lorem Ipsum. It's OK if your Lorem text is different than what is pictured here, so feel free to use one of the more creative generators!

Now add style rules to accomplish the following:

  • The image should be floated to the right, and by default it should have a maximum height of 300px, and a left margin of 1em.
  • The image should have rounded edges with a radius of 5px.
  • On screens 769 pixels and wider, the maximum height should increase to 400px.
  • The background color of this section should be #dbdce3.

Footer

The last part of this page is the footer. Add an element that is the most semantically-appropriate for page footer content.

The layout for the footer is a responsive, multi-column one. By default, the first block (the Dawg Coffee copyright declaration) and the second block (the icons attribution) should be stacked on top of each other. On screens 768 pixels and wider, they should layout side-by-side. The first block should consume 3 Bootstrap grid units and the second block should consume 9 grid units.

The copyright declaration in the first block uses the copyright symbol. The easiest way to include this is to use the HTML symbol entity for the copyright sign.

The content in the second block is an attribution that is required by the license attached to the SVG icons we used. Copy and paste this content into that block:


<p>Icons by <a href="http://www.freepik.com" title="Freepik">Freepik</a>, <a href="http://www.bogdanrosu.com" title="Bogdan Rosu">Bogdan Rosu</a> from <a href="http://www.flaticon.com" title="Flaticon">www.flaticon.com</a> is licensed under <a href="http://creativecommons.org/licenses/by/3.0/" title="Creative Commons BY 3.0">CC BY 3.0</a></p>

Now add style rules to accomplish the following:

  • The footer should have a border on top, 3px tall, solid, in the color #D87E28.
  • The background color for the footer should be #afb3c2.
  • The font size should be 12px.
  • There should be 1em of padding on the top.
  • On screens 768 pixels and wider, the second block (the icons attribution) should be right aligned.

Build and Style the Orders Page

Lastly, create a placeholder page for the online ordering page. Create another HTML page in the same directory as your index.html named order.html. This page should contains the same basic structure as your home page, as well as the navigation bar. The header content is similar, but slightly adjusted (see pictures). There should be a line break between "Coffee" and "Orders" in the top-level heading. After the header there should be just a simple "coming soon..." paragraph inside a Bootstrap container. We will build this page in a later challenge.

The links in the navigation bar need to be adjusted so that all besides the 'order' link refer back to the index.html page, or a section within it. Click the 'order' link should scroll to the top of the current page.

Advanced Optional Steps

If you found the required steps easy, or if you're wanting some extra challenge, try one or more of these optional advanced steps. You will receive extra credit for these based on the difficulty of the step and quality of your implementation.

Animated Transitions For Navigation Bar Hyperlink Hover Styles

Both you and the Bootstrap library added style rules that change the formatting of hyperlinks when you hover over them, but these formatting changes happen instantly. Use the CSS transition property to animate these changes over a short period of time (e.g., 0.3 seconds).

Generate Your CSS Using Sass

You may have noticed several things that were repeated in a few different style rules, especially color values. In larger projects this repetition can become extreme, and changing your colors can become quite a chore. To make this more tolerable, most professional developers use a CSS preprocessor tool, the most popular of which is known as Sass.

For this step, rewrite your main.css file in the SCSS language, taking advantage of variables, nesting, and functions. Then use the sass command line tool, to generate your main.css file from your main.scss file. While you're working on this, you will probably want to use their feature that watches your source SCSS file and automatically regenerates the target CSS whenever the source SCSS file changes.

Use variables for all your colors. For the darker background on the cafe info and story sections, use the Sass darken() function to darken the page background by 5%. For the footer background color, darken the page background by 20%.

Also use a variable for the set of fall-back fonts that are used if Lato and Raleway cannot be downloaded from the Google fonts server.
