Mobile first: desinging with Bootstrap

Summary

The essential HTML and CSS skills you practiced and mastered in the previous challenge enable you to build attractive web pages, but those pages are still fairly limited in terms of page layout. Professionally-designed pages typically use more sophisticated layout techniques, such as multiple side-by-side columns and absolute positioning.

Professionally-implemented web pages are also increasingly "responsive," meaning they respond to different screen sizes by adjusting layout, font sizes, graphics, etc. The same responsive page can look fantastic and deliver a great user experience on everything from a small phone screen to a giant high-definition monitor.

In this challenge you will practice and master the skills necessary to create complex and responsive page layouts. To make our lives a little easier, we will use a responsive CSS framework called Bootstrap, which was developed by the good folks at Twitter. Bootstrap defines a bunch of CSS classes for us that make building complex and responsive layouts much easier. It also applies a bunch of default formatting to the standard HTML elements based on the principles of good graphic design.

Objectives

By completing this challenge, you will practice and demonstrate mastery of the following skills:

  • Adding the Bootstrap CSS framework to your web pages to automatically improve the graphic design
  • Using the Bootstrap grid system to create multi-column page layouts
  • Using media queries to adjust page layout and other formatting based on the screen size
  • Using relative and absolute positioning to place elements exactly where you want them

Fork and Clone the Repo

Just as you did in the previous challenge, sign in to GitHub and fork my layout challenge repository into your own account. Then clone your forked repo to your local machine (not my original one).

After you've cloned, copy over the index.html and css/main.css files you built for the previous challenge into this new repository. We will edit the HTML file a bit, and change the styles so that we get different layouts on differently-sized screens.

The HTML Page

Open your index.html file (the one you copied over from the previous challenge) in your editor, and make the following changes:

  • Remove the link to the css/normalize.css stylesheet, and replace it with a link to the Bootstrap minified stylesheet, which you can load directly from the Bootstrap Content Delivery Network (CDN). Bootstrap already includes all the normalize styles at the top of their stylesheet. Make sure the Bootstrap one appears before your css/main.css so that we can adjust some of the Bootstrap styles.
    When adding style sheet links using absolute URLs (e.g., those pointing to a CDN and not your web server), it's a good idea to use the https:// protocol instead of http://. Browsers will often show errors or fail to load resources from HTTP URLs when the original page itself was loaded from an HTTPs URL. If you omit the protocol altogether (i.e., the URL starts with //), the browser will use the same protocol as it used for the original page, but if you load that page from the file system (i.e., you double-click the file, or use "Open In Browser" from your editor), the protocol will be file://, which won't work with a CDN (the CDN isn't in your local file system). The easiest way to make sure you are covered in every scenario is simply to always use HTTPS for these absolute URLs to extra resources.
  • Add the following navigation links to your page header section, just after the 'Feeding Hungry Huskies Since 2013' paragraph: 
    <nav role="navigation">
    <ul>
        <li><a href="#location">location</a></li>
        <li><a href="#menu">menu</a></li>
        <li><a href="#about">about</a></li>
    </ul>
</nav>
    It's customary now to use <ul> and <li> elements nested within a <nav> element for navigation links. The navigation links are a set of related items, hence the <ul> and <li> elements, and <nav> is the standard HTML 5 element for navigation content. The role="navigation" attribute is used by screen readers and other assistive technologies to find the navigation links on the current page.
  • Add the necessary elements and predefined Boostrap style classes to produce a 3-column layout in the restaurant information section, and a 2-column layout in the menu section on medium screens and wider, as shown in the screen shots above (hint: you'll want to use the col-md-* class names so that the columns are side-by-side on medium to large screens). The columns should collapse to be stacked on top of one another on screens narrower than 992 pixels. Remember that Bootstrap grid rows must exist within an element that has either the container or container-fluid style class. You want to use container-fluid, and the logical place to add that style class would be your main page content element.

Your Stylesheet

Now open your css/main.css file in your editor. Our previous styles were great for a large screen, but we want to separate those now into two groups: those that apply to all devices, including small phone screens; and those that should apply only when the screen width is 992 pixels or wider. You can use a media query to segment your styles into these two sets.

Formatting for All Devices

Most of the style rules you created before still apply to all devices, but we need to make a few changes in order to get the different layout you see in the small-device screenshot above:

  • The background image on the body should be applied only when the width is 992px and wider; narrower devices should not have a background image, as the content will fill the entire width of the screen
  • The margins you used to apply to the main page content element should be applied only when the width is 992px or wider; on narrower devices, the page content should fill the entire width of the screen
  • The font size on the body should be reduced to 16px (we will increase it to 18px on larger screens)
  • The Bootstrap .container-fluid rule will apply a top and bottom margin to the element, but we want that element to be right up against the top of the page on small screens, so create a rule with the same class selector and set the top and bottom margins to 0
  • The tag line ('Feeding Hungry Huskies Since 2013') should be displayed only when the width is 992px or wider; on narrower screens that element should not be displayed
  • The <ul> element within the <nav> element should have padding and margins of 0 on all sides
  • The <li> elements within the <nav> element should have no bullets, should be displayed as inline-block elements, and should have a right margin of 1em
  • The navigation hyperlinks themsleves should have the color #55E, but that should change to #00E when the readers hovers the mouse over or sets focus to them

Formatting for Medium and Large Screens (992px and Wider)

When the screen size is 992px or larger, apply the following additional formatting:

  • As was noted above, the body background image should be applied for these sized screens, in exactly the same way as you did for the previous challenge
  • Also as noted above, the tag line should be displayed for these sized screens
  • The body font size should also be increased to 18px
  • The Bootstrap .container-fluid rule should be altered to set the left margin to 30%, and the top, right, and bottom margins to 20px
  • The entire <nav> element should be positioned 10px from the top and right of the page header element

Review Your Page

You've now added all the content and style rules to make your page look like the screen shots. Look at it in the browser and if something doesn't look correct, review that part of the page and style rules. Remember, it doesn't have to be pixel-perfect, but it should be close, and should respond to the screen width as described.

If everything looks and behaves correctly, you now know how to build responsive web pages with complex layouts—congratulations! You could definitely start building interesting web sites for clients.
