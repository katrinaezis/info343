Summary

Objectives

By completing this challenge, you will practice and demonstrate mastery of the following skills:

  • Building a web-page with multi-page architecture
  • Designing a data-driven web-page with Angular.js
  • Build a mobile-friendly web-page using modern web-standards (i.e., bootstrap).
  • Practice your design skills to impress future collaborators.

Data-driven content

At this point in the course, you should understand the power of data-driven content. Rather than write out each element in HTMl, your content should be derived from data, and presented using angular directives such as ng-repeat. You have a few different options for storing and accessing your data

  • Store your data (i.e projects, links, descriptions, etc.) in a .csv file. If you do this, you'll either need to:
    • Parse the data once you've loaded it, which is far easier if you use a library like PapaParse
    • Write your .csv file, then manually turn it into a JSON object using a service like Mr. Data Converter
  • Use a service like Parse.com. How you get the data there is a question you'll have to solve....

This loaded data should drive the content of the page. For example, if you are showcasing multiple projects you've worked on (including images, a desription, etc.) your data should maintain those images/descriptions/links, and you should use angular to handle the display of that data. You could even go as far as storing other contact (i.e., contact info, land page description) in data as well.

Multi-page Architecture

Building pages with multi-page architecture is a common web-standard. Rather than figuring out page-redirects with jQuery, it is much more efficient to handle these component with the Angular Route service. This allows users to create and load html fragments into their page content without reloading the page. In order to do this, you'll need to perform the following tasks:

  • Create a navigation element that persists across pages (I suggest you use bootstrap nav elements).
  • Create at least 3 different pages that users can navigate across. These could include a home page, projects page, interests page, contact page, etc.
