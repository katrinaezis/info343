Teaching Presentation

Summary

For the first several weeks of class, you've been led through interactive lectures about how to use different JavaScript libraries and packages -- now it's your turn to do the teaching. To truly master a topic, you must be able to both implement it, and explain it to others. In this challenge, you'll identify a new skill of your choice, and give a 25 minute presentation on it, including an interactive exercise.

The beauty of the open-source community is that there are millions of people building tools that you can use to solve your problems. In your teaching presentation, you should identify a robust, well documented, and useful set of code to share with your peers. Here are some ideas:

  • Introduce a new JavaScript library (Lazy.js)
  • Teach a new angular module (translation, D3 in Angular)
  • Introduce the class to a new framework (react.js, backbone.js, knockout.js, etc.)
  • Introcude a new skill such as server-side scripting with Node.js
Note, you should not present a library that solves a single problem. Instead, find a tool that people can use in a variety of contexts. Twenty-five minutes goes fast, so you'll have to be concise.

Objectives

By completing this challenge, you will practice and demonstrate mastery of the following skills:

  • Identify and teach yourself a new JavaScript library
  • Develop teaching materials for explaining a topic
  • Evaluate the limitations of a library
  • Code up an exercise that both highlights the functionality of a library, and teaches it

Resources

This challenge is really about going out there and finding a library that you find interesting/fun/useful. Try and find a solution that someone else has made to a problem you've encountered, or build a creative application using the tools that you find!

Required Features

Depending on the specifics of the library you choose, your presentation may take on different formats. Make sure to pick a moderately sophisticated library or package that is actually worth learning for you and others. Regardless of the platform you use, you must cover the following information

  • What does the package do (i.e., why would you use it)?
  • Explain the syntax of the package (i.e., how does it work)?
  • What challenges did you encounter in using the package, and/or why would you not use it?

In addition to communicating the information above to your fellow classmates, you must include an interactive exercise that allows students to explore the implementation of the package. Feel free to use the approach in class (jsBin), or another method, such as coding together, or providing a coding challenge. This exercise should demonstrate how to use the package, and in doing so, highlight its functionality

You may choose to divide your time (25 minutes, total) as you see fit. If you believe more explanation is needed, spend more time on describing how the package works. If you would rather create an involved and well documented exercise for your classmates, allocate more time to the interactive component of your assignment.

All group members must participate in presenting the material to the classroom, as well as in the preparation of the materials. If there are any concerns with the distribution of work, try to work it out with your fellow group members, or involve Mike or you TA if necessary.
